# Account recovery без email и телефона

Friendscape принципиально не использует email и телефон как обязательные идентификаторы. Поэтому восстановление доступа строится на внутренних и локальных способах подтверждения.

## Что реализовано

### 1. Recovery-фраза

Пользователь может создать recovery-фразу в настройках безопасности. Фраза показывается один раз, после чего на сервере хранится только bcrypt-хеш.

Маршруты:

- `POST /api/auth/security/recovery-phrase` — создать/перевыпустить фразу после проверки текущего пароля.
- `POST /api/auth/recover/phrase` — восстановить вход по имени, фамилии и recovery-фразе.

Особенности:

- email/phone не используются;
- фраза нормализуется перед проверкой;
- повторно посмотреть фразу нельзя, только перевыпустить;
- перевыпуск сразу делает старую фразу недействительной.

### 2. Доверенное устройство + PIN

Устройство не становится доверенным сразу. Оно получает доверие после нескольких сессий по существующей DFSN/Device-логике. Пользователь может задать PIN для доверенного устройства.

Маршрут восстановления:

- `POST /api/auth/recover/trusted-device`

Проверяется:

- имя и фамилия пользователя;
- fingerprint текущего браузера;
- статус `trusted` у устройства;
- наличие PIN;
- bcrypt-сравнение PIN.

### 3. Резервные коды

Существующая схема резервных кодов сохранена. Код одноразовый: после успешного восстановления соответствующий хеш удаляется из массива.

### 4. Секретный ответ

Существующий секретный ответ сохранён как дополнительный способ. Он не требует внешних контактов, но теперь не является единственным основным способом.

### 5. Passkey foundation

В Prisma добавлена модель `AccountPasskey` для будущего WebAuthn/passkey flow:

- `credentialId`;
- `publicKey`;
- `counter`;
- `transports`;
- `label`;
- `disabledAt`;
- `lastUsedAt`.

Полный браузерный WebAuthn challenge/assertion flow пока не включён в UI, потому что его нужно делать отдельным проходом с аккуратной криптографической проверкой и UX.

## UI

Добавлены страницы:

- `/recover/phrase`
- `/recover/trusted-device`

Страница `/forgot-password` теперь предлагает новые способы восстановления.

В `/settings` добавлены:

- статус recovery-фразы;
- количество доверенных устройств с PIN;
- создание/перевыпуск recovery-фразы.

## Prisma

Добавлено поле:

```prisma
User.recoveryPhraseHash String?
```

Добавлена модель:

```prisma
AccountPasskey
```

npx prisma generate
