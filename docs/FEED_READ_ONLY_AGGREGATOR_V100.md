# v100 — Feed read-only aggregator

Цель прохода: убрать создание поста из ленты и оставить ленту как спокойный агрегатор материалов.

## Сделано

- Удалён composer `Что нового?` из `/feed`.
- Удалён runtime публикации из `app/feed/page.jsx`: локальный черновик, open-state, posting-state и POST в `/api/feed/posts`.
- Обновлён hero-текст: лента больше не обещает публикацию записей из этого экрана.
- Оставлены чтение, фильтры, поиск, stories rail, комментарии, реакции, сохранение, жалобы и внутренний шеринг.
- Feedback entry остаётся в настройках, без плавающей кнопки в нижней навигации.

## Логика продукта

Лента теперь работает как агрегатор: пользователь читает материалы, реагирует и обсуждает. Создание контента должно жить в более осмысленных местах: профиль, сообщество, отдельный create-flow.
