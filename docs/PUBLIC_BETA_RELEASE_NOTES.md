# Friendscape Public Beta Release Notes

Статус: public beta candidate после v97.

## Что готово для beta-тестеров

- Регистрация без email и телефона.
- Вход по паролю и passkey flow.
- Recovery phrase / backup-коды / trusted devices.
- Профиль и настройки приватности.
- Лента реальных постов.
- Комментарии, реакции, сохранение, жалобы.
- Мессенджер с сообщениями, заявками, media flow и базовой безопасностью.
- Сообщества: создание, вступление, заявки, посты, участники, media.
- Уведомления.
- Поддержка и обращения.
- Feedback/report bug flow через раздел поддержки в настройках и страницу `/feedback`.
- Публичные trust/legal страницы.
- Production runbooks: deploy, backup, restore, release, rollback.

## Известные ограничения

- Beta может иметь UI-шероховатости на отдельных Android/iOS браузерах.
- Полная нагрузочная проверка на больших данных ещё должна пройти после сбора первых beta-метрик.
- Realtime рассчитан на Postgres transport; memory transport оставлен только для dev/local.
- Object storage должен быть production-grade и приватным, local uploads допустимы только как осознанный single-server fallback.
- DFSN остаётся закрытой технологией: пользователю объясняются принципы защиты, но не внутренние сигналы, веса, пороги и эвристики.

## Как отправлять feedback

Отзыв теперь отправляется из настроек: `Настройки → Поддержка → Оставить отзыв`. Через этот flow можно отправить:

- баг;
- неудобный сценарий;
- проблему первого запуска;
- проблему скорости;
- жалобу на поведение.

Не нужно отправлять пароли, recovery-фразы, backup-коды или любые секреты.

## Минимальный smoke перед публикацией

1. Новый пользователь регистрируется без email/phone.
2. Пользователь создаёт первый пост из профиля или сообщества.
3. Пользователь находит человека.
4. Пользователь открывает первый чат.
5. Пользователь создаёт первое сообщество.
6. Пользователь отправляет beta feedback.
7. Админ видит тикет и может сменить статус.

## Release decision

Public beta можно публиковать только после успешных команд:

```bash
npm run verify:launch
npm run qa:alpha
npm run beta:qa
npm run security:check
npm run release:prod:strict
npm run build
npm run smoke:e2e
```
