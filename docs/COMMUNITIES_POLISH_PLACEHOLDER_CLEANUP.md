# Communities polish + placeholder cleanup

В этом проходе раздел сообществ доведён до более рабочего состояния без декоративных заглушек.

## Что изменено

- Заменены сырые текстовые пустые состояния на аккуратные `EmptyState`-карточки.
- Убраны формулировки вида “пока / фейковые карточки” из UI сообществ.
- Для списка сообществ добавлены понятные состояния:
  - загрузка;
  - ничего не найдено;
  - у пользователя нет своих сообществ;
  - нет доступных сообществ.
- На странице сообщества добавлены рабочие пустые состояния для:
  - ленты;
  - правил;
  - участников;
  - заявок;
  - очереди модерации;
  - инвайтов;
  - журнала действий.
- Вкладки управления показывают счётчики заявок и жалоб.
- Жалоба на пост больше не уходит мгновенно общей причиной: добавлена модалка с выбором причины и комментарием для модераторов.
- Добавлены стили для новых loading/empty/report-состояний.

## Что не трогалось

- Prisma-схема не менялась.
- API контракты сообществ не менялись.
- Логика ролей, заявок, инвайтов и модерации не менялась.

## Проверки

- `node --check` для серверных файлов.
- Поиск временных формулировок в `app/communities`.

## Следующий рекомендуемый шаг

Следующий проход: связать сообщества с общей лентой и профилем глубже:

- фильтр “посты из сообществ” в feed;
- карточка сообщества в посте;
- переход из поста в сообщество;
- настройки видимости сообществ в профиле;
- проверка прав просмотра community-постов в общей ленте.
