# Settings security modal cleanup

Этот проход убирает системные `window.prompt` / `window.confirm` из чувствительных действий в настройках безопасности.

## Что изменено

- Отключение passkey теперь подтверждается внутренней модалкой с вводом текущего пароля.
- Установка/смена PIN доверенного устройства теперь идёт через внутреннюю модалку: PIN + текущий пароль.
- Отключение доверенного устройства теперь подтверждается внутренней модалкой с предупреждением и текущим паролем.
- Системные браузерные окна больше не используются в `app/settings/page.jsx`.

## Зачем

- Единый UX без нативных prompt/confirm.
- Меньше случайных подтверждений.
- Чувствительные действия выглядят как часть интерфейса Friendscape.
- Философия сервиса без email/phone сохраняется: подтверждение идёт через пароль, passkey, PIN и доверенные устройства.

## Проверка

```bash
node scripts/audit-placeholders.mjs
```

Дополнительно можно проверить:

```bash
grep -n "window\.prompt\|window\.confirm" app/settings/page.jsx
```

Результат должен быть пустым.
