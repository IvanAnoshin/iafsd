# Feed/posts stabilization

## Что изменено

- Убрано runtime-создание demo-постов из `/api/feed`.
- `/api/feed` теперь отдаёт реальные посты профилей и сообществ с учётом связей пользователя.
- Добавлена публикация личного поста из ленты через `POST /api/feed/posts`.
- Добавлены редактирование и мягкое удаление личного поста через `/api/feed/posts/[postId]`.
- У личных постов появилась проверка видимости: `public`, `friends`, `followers`, `private`.
- Проверки доступа применяются к прямому открытию поста, feed-выдаче и существующим действиям через `canViewerAccessPost`.
- На странице ленты добавлен минималистичный composer без декоративных заглушек.
- В меню поста добавлены действия владельца: изменить и удалить.
- Профильная публикация теперь также сохраняет `visibility` и может попадать в общую ленту при наличии прав.

## Правила видимости

- `public` — видят все авторизованные пользователи.
- `friends` — видят друзья и автор.
- `followers` — видят подписчики, друзья и автор.
- `private` — видит только автор.
- `community` — используется для постов сообществ.
