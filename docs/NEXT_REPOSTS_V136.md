# NEXT REPOSTS V136

## Что сделано
- В Next-проект добавлена рабочая модель репостов через `Post.repostOfId`.
- Добавлен API `POST /api/posts/[id]/repost`.
- Репосты защищены от дублей: один пользователь не плодит одинаковые репосты одного оригинала.
- Репост репоста автоматически разворачивается до исходной публикации, без визуальной матрёшки.
- Репост создаётся как обычный пост пользователя с `type: "repost"` и вложенным `repost_of`.
- Оригинальный пост получает счётчик `profile_reposts` в payload.
- При удалении репоста счётчик оригинала уменьшается.
- В ленте и профиле добавлен общий компонент `PostRepostPreview` для компактной вложенной карточки оригинала.
- Bottom-sheet шеринга в ленте получил действие `Репостнуть в профиль`.
- В профиле и публичном профиле действие `Поделиться` теперь делает рабочий репост через подтверждение.
- Публичный профиль теперь показывает не только текстовые посты, но и медиа/репосты.
- Success-баннеры для обычных действий репоста не добавлялись.

## Важно после распаковки
Так как изменилась Prisma-схема, локально нужно выполнить миграцию/синхронизацию базы и пересобрать Prisma Client.
В dev-режиме для текущего проекта обычно достаточно штатного проекта процесса Prisma. Если запускаешь руками:

```bash
npx prisma db push
npx prisma generate
npm run build
```
