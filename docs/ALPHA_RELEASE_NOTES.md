# Friendscape Alpha Release Notes

Версия: v87 — Final Alpha QA + Bugfix
Статус: готово к закрытой альфе для небольшой группы доверенных тестеров.

## Что готово для тестирования

- Регистрация без email и телефона.
- Вход по паролю, recovery-фраза, backup-коды, passkey/WebAuthn и trusted devices.
- DFSN-сигналы для безопасности без превращения email/phone в обязательные идентификаторы.
- Лента с реальными постами, комментариями, голосами, сохранением и шерингом.
- Профиль, приватность профиля, медиа-профиля и связи пользователя.
- Мессенджер: список чатов, заявки, отправка сообщений, медиа, голосовые/видео-заметки, реакции, жалобы, удаление.
- Сообщества: создание, вступление, заявки, посты, участники, медиа, инвайты и базовая модерация.
- Уведомления и mark-all-read.
- Глобальная очередь модерации и audit log модераторских действий.
- Production env/deploy checklist, backup/restore/cleanup ops.
- Минимальные smoke/e2e сценарии.

## Что тестерам можно делать

- Зарегистрироваться без email/phone.
- Проверить вход/выход и восстановление доступа.
- Создавать посты, комментировать, голосовать, сохранять и удалять свой контент.
- Создавать чаты, принимать заявки, отправлять сообщения и медиа.
- Создавать сообщества, приглашать людей, модерировать заявки и контент внутри сообщества.
- Отправлять жалобы на посты, комментарии, сообщения, профили и сообщества.
- Проверять настройки приватности, passkey, устройства и recovery-механики.

## Известные ограничения альфы

- Realtime всё ещё рассчитан на один Node-процесс; масштабирование запланировано на v91.
- Production media hardening будет усилен в v92: signed URLs, quotas, MIME sniffing, EXIF cleanup.
- Monitoring/logging/alerting запланированы на v90.
- Legal/trust pages и data export/delete account будут добавлены в v88–v89.
- UI уже пригоден для тестирования, но финальная responsive/accessibility-полировка запланирована на v94.
- Закрытая альфа не означает публичный запуск: тестеры должны ожидать баги, edge cases и ручную обратную связь.

## Как сообщать баги

Для закрытой альфы основной путь — раздел поддержки в настройках. Если он недоступен, баг можно передать владельцу проекта напрямую с такими данными:

1. Что делал пользователь.
2. На какой странице это произошло.
3. Что ожидалось.
4. Что случилось фактически.
5. Устройство, браузер, ширина экрана.
6. Скриншот или короткое видео, если есть.

## Перед выдачей тестерам

Рекомендуемый локальный/стейджинг-чек:

```bash
npm run verify:launch
npm run qa:alpha
npm run audit:placeholders
npm run audit:access-control
npm run audit:sensitive-routes
npm run smoke:e2e
```

`smoke:e2e` требует поднятый сервер, рабочую базу и тестовое окружение.
