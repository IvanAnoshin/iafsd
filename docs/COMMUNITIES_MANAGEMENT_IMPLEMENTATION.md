# Communities Management Pass

Добавлен управленческий слой для сообществ без email и телефонов.

## Что реализовано

- PATCH `/api/communities/[slug]` — редактирование настроек сообщества.
- PATCH `/api/communities/[slug]/members/[memberId]` — смена роли участника.
- DELETE `/api/communities/[slug]/members/[memberId]` — удаление участника из сообщества.
- GET `/api/communities/[slug]/moderation` — журнал действий сообщества.
- UI во вкладке `Настройки`:
  - название;
  - описание;
  - город;
  - тип сообщества;
  - права публикации;
  - видимость участников;
  - тема аватара;
  - теги;
  - правила;
  - заявки;
  - инвайт-коды;
  - discoverable.
- UI во вкладке `Участники`:
  - текущая роль;
  - смена роли;
  - удаление участника.
- UI во вкладке `Заявки`:
  - сообщение пользователя;
  - дата заявки;
  - принятие/отклонение.
- Журнал действий в настройках.

## Правила безопасности

- Настройки меняют только `owner` и `admin`.
- Роли меняют только `owner` и `admin`.
- `admin` не может менять владельца или другого администратора.
- Нельзя удалить владельца.
- Нельзя удалить самого себя через панель управления участниками.
- Приватные сообщества не показываются в поиске.
- Закрытые и приватные сообщества всегда требуют заявку на вступление.
- Видимость списка участников учитывается на backend.

## Следующий логичный проход

Community Moderation:

- скрытие/удаление постов сообщества;
- мут участника;
- бан участника;
- жалоба на пост/участника внутри сообщества;
- очередь модерации;
- видимые действия модератора в UI.
