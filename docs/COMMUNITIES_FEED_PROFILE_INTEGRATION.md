# Communities Feed + Profile Integration

## Что сделано

- В общей ленте появилась отдельная вкладка `Сообщества`.
- Настройки ленты теперь умеют включать/выключать вкладку сообществ.
- Посты сообществ получили отдельную карточку-ссылку внутри feed-карточки.
- По клику на карточку сообщества пользователь переходит на `/communities/[slug]`.
- Поиск по ленте теперь ищет не только по тексту поста и автору, но и по названию/slug сообщества.
- Backend ленты больше не отдаёт community-посты закрытых/приватных сообществ посторонним пользователям.
- Прямые API для открытия поста (`/api/posts/[id]`, `/api/feed/posts/[postId]`) проверяют права просмотра community-постов.
- Публичные community-посты видны в feed только если сообщество публичное.
- Посты закрытых/приватных сообществ видны в feed только активным участникам.

## Почему это важно

Раньше community-посты были уже связаны с сообществами, но в общей ленте не было отдельного пользовательского режима и строгой проверки на уровне feed API. Теперь связь стала полноценнее: лента понимает, что материал пришёл из сообщества, UI показывает источник, а backend не раскрывает приватные community-посты.

## После установки

```bash
npx prisma generate
npx prisma db push
npm run dev
```

`db push` нужен, потому что в `UserFeedSettings` добавлено поле `showCommunities`.
